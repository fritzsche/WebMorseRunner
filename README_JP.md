# Web Morse Runner

ウェブブラウザで動作する CW コンテストシミュレーターです。

## 概要
これは Morse Runner を Web アプリケーションとして新規に書き直したバージョンです。

アプリは GitHub Pages で公開されています: https://fritzsche.github.io/WebMorseRunner/

**Translations:** [English](README.md) · [Deutsch](README_DE.md)


このプロジェクトは [VE3NEA - Alex Shovkoplyas 氏による Morse Runner](https://github.com/VE3NEA/MorseRunner) プログラムに触発されたものです。Web アプリケーションとして全新しく開発しようと思ったきっかけは、著者が以前行った [Morse Runner の Linux および Mac への移植](https://github.com/fritzsche/MorseRunner) の経験からです。


このプロジェクトは新規かつ Alex のプロジェクトとは独立しています。オリジナルの Morse Runner は Windows 向けで Pascal で開発されていますが、このバージョンは JavaScript でゼロから完全に書き直され、ウェブブラウザでの使用を目指しています。


## 機能

* JavaScript を使用しブラウザで動作
* プライバシー: アプリはあなたのブラウザで実行されます。Web Morse Runner は一切データを収集せず、ウェブサーバーにも送信しません。
* AudioWorklet を使用した Web Audio API
* バックエンド不要
* コンテストモード:
    * Single Call (シングルコール)
    * Pileup (パイルアップ)
    * WPX Contest (WPX コンテスト)
    * DARC CW Ausbildungscontest (CWA)
    * A1CLUB AWT
    * CWOps CWT
    * IARU VHF
* (複雑な) ノイズ
* 変調
* フィルタ
* AGC
* calls.txt からコールサインリストを読み込み
* WPM での速度設定
* QSO/h レートとチャートの表示
* 設定:
    * Pitch (ピッチ)
    * RX Bandwidth (受信帯域幅)
    * Monitor Level (モニター音量)
    * RIT
    * コンテストの継続時間
    * My Call (自局コールサイン)
* ボタンおよびファンクションキーによる送信
* ESM (Enter to Send Message)
* スコアの計算と表示
* ログの表示
* QSO の確認
* バンドコンディション (QRM/QRN/QSB/Flutter/LID)
* トランスクリプト
* WAV 録音
* エキスパート設定:
    * Farnsworth タイミング
    * DX 局の WPM (標準または個別の min/max 範囲)
    * コンテスト開始オフセット (進行中のコンテストへの参加をシミュレート)

## 使い方
Web Morse Runner はコンテストシミュレーターです。目標はできるだけ多くのポイント (QSO) とマルチ (プリフィックス) を獲得することです。

まず自局のコールサイン、希望する WPM などのパラメータを設定し、Run ボタンを押してコンテストを開始します。

**Single Call** モードでは局があなたを呼び出しますので、Call/NR フィールドに QSO をログとして記録する必要があります。**Pileup** モードでは CQ を出す必要があります。"Activity" パラメータに応じて、応答してくる局の数が変わります。

それぞれのボタンまたはファンクションキーを使用して局に応答します。使用するキーボードについては [Keyboard](#keyboard) セクションを参照してください。QSO を終了するには TU を送信する必要があります。

シミュレーターは ESM (Enter to Send Message) をサポートしており、Enter キーを押すと QSO の状態に応じたメッセージを送信できます。ESM の詳細は [こちら](#esm-enter-to-send-message) を参照してください。

現実の世界と同様に、送信中の局はあなたが送信している内容を聞いていません。したがって、返信する前に DX 局の送信が終わるまで待つ必要があります。

## プライバシー
Web Morse Runner はあなたのブラウザで実行されます。アプリ起動時に必要な静的ファイルがサーバーから読み込まれますが、**いかなる利用データも収集されません**。

## ホスティング
Web Morse Runner はスタンドアロンの JavaScript アプリで、公式版は GitHub でホスティングされています。
独自の (ローカル) ウェブサーバーでプロジェクトをホスティングすることもできます。Web Morse Runner を実行するには "Secure Context" (https または localhost) が必要です。

## 対応 OS
Web Morse Runner は **Mac** / **Windows 11** および **Linux** で動作確認が取れています。
メインのブラウザは **Chrome** ですが、**Edge**、**Firefox**、**Safari** でも動作することが報告されています。
変更ごとにすべての OS とブラウザの組み合わせをテストすることはできないことをご了承ください。

*iOS での使用*: 著者は Safari と Chrome を使ってさまざまな iOS デバイス (iPhone/iPad) で Web Morse Runner を動作させることができました。Web Morse Runner を使用するには、外部の Bluetooth キーボードを接続することができます。
音声を聞くには、マナーモードがオフになっており、音量が設定されていることを再確認してください。バッテリー設定でコンテストが中断されないことを確認してください。ファンクションキーの代わりに、Web Morse Runner 内の F1-8 ボタンを押すとセッションがアクティブのままになります。


## 設定
Web Morse Runner で行った設定は、ブラウザのローカルストレージに保存されます。

* **Call**: あなたの自局コールサイン。
* **QSK**: 選択すると、送信と受信を超高速に切り替えるため、dit/dah の合間にバンドアクティビティを聞くことができます。
* **CW Speed**: あなたの送信速度です。すべての呼び出し局はこれより遅い速度で応答します。
* **Pitch**: サイドトーンの周波数。
* **RX Bandwidth**: 適用されるフィルタの帯域幅。この値を小さすぎるとフィルタ範囲外の呼び出し局を聞けなくなる可能性があります。
* **Mon. Level**: 自分のサイドトーンの音量です。全体の音量はシステムのオーディオレベルを使用してください。
* **RIT**: RIT の値。スライダーを動かして RIT を周波数方向に上下に移動します。

## バンドコンディション

* **QRM**: 他の運用中の局からの干渉が時折発生します。
* **QRN**: 静電気による干渉。
* **QSB**: 信号強度が時間とともに変化します。
* **Flutter**: 一部の局はオーロラのような音を発します。
* **LIDS**: 一部の局は、あなたが他の局と交信中に呼び出してくる、符号を送る際に間違える、メッセージを誤ってコピーする、599 以外の RST を送ってくるなどの行為を行います。

## コールサインのアップロード
リリース 0.05 以降、Web Morse Runner は calls.txt に保存されているコールサイン情報をブラウザのローカルストレージにキャッシュします。calls.txt ファイルは、ローカルストレージをクリアしたとき、または **reload** リンクを押したときに再読み込みされます。
**upload** リンクから、ファイルアップロードで独自のコールサインを提供することもできます。
読み込むファイルは、1 行に 1 コールサインを含む標準的なテキストファイルである必要があります。
アップロード形式は将来変更される可能性があることにご注意ください。

コールサインファイルの形式はとてもシンプルです。サンプルは GitHub リポジトリ (Example_Calls フォルダ) にあります。


## コンテストモード

Web Morse Runner は以下のコンテストモードをサポートしています:
* **Single Call**: 常に 1 局があなたを呼び出します。パイルアップは無く、CQ を呼ぶ必要はありません。
* **Pileup**: このモードでは、局が応答する前にまず CQ を呼ぶ必要があります。*Activity* パラメータが、平均で何局が応答するかを決定します。呼び出し中の局の数はランニングクロックの上に表示されます。
* **WPX Contest**: Pileup モードと同様ですが、パイルアップの局数は表示されません。バンドコンディション (QRM/QSB など) の使用は強制されません。
* **HST** (HST = High Speed Traffic): 局が次々と素早く呼び出します。*Activity* パラメータで現れる局の数を制御します。各 QSO は ESM でログします。
* **DARC CWA** (EXPERIMENTAL): 交換情報は DOK です。局に必要な DOK 情報を含む [DL-All_DOK.txt](https://raw.githubusercontent.com/fritzsche/WebMorseRunner/refs/heads/main/Example_Calls/DL-All_DOK.txt) ファイルを [アップロード](#upload-calls) する必要があります。現時点ではポイントは DOK ではなくプリフィックスで計算されますが、今後更新される可能性があります。
* **AWT** (EXPERIMENTAL): 交換情報は名前です。名前を含むサンプルコール [AWT.txt](https://raw.githubusercontent.com/fritzsche/WebMorseRunner/refs/heads/main/Example_Calls/AWT.txt) を [アップロード](#upload-calls) する必要があります。ポイントの計算は依然としてプリフィックスに基づいています。これは今後追加される可能性があります。
* **CWOps CWT** (EXPERIMENTAL): 交換情報は名前と CWOps ナンバーです。名前とナンバーを含むサンプルコール [CWOps.txt](https://raw.githubusercontent.com/fritzsche/WebMorseRunner/refs/heads/main/Example_Calls/CWOps.txt) を [アップロード](#upload-calls) する必要があります。ポイントの計算は依然としてプリフィックスに基づいています。これは今後追加される可能性があります。
* **IARU VHF**: 交換情報は RST、シリアルナンバー、および自分自身の 6 文字の QTH ロケータ (LOC, 例: `JN58TD`) です。新しいサンプルファイルの一つを [アップロード](#upload-calls) する必要があります — 混合モードの局向けには [`IARU-VHF-NOCW.txt`](https://raw.githubusercontent.com/fritzsche/WebMorseRunner/refs/heads/main/Example_Calls/IARU-VHF-NOCW.txt)、CW 運用局向けには [`IARU-VHF-ONLYCW.txt`](https://raw.githubusercontent.com/fritzsche/WebMorseRunner/refs/heads/main/Example_Calls/IARU-VHF-ONLYCW.txt) — これらのファイルにはコールサインと対応する QTH ロケータが含まれます。コンテスト開始前に My Exchange パネルの **LOC** フィールドに自分自身の QTH ロケータを設定してください。


## パイルアップ
パイルアップをマスターするには、1 局ずつ拾うのが効果的です。通常、最も明瞭にコピーできる周波数で呼び出している局から始めるのが良いでしょう。
1 局をピンポイントで選ぶには疑問符を使用できます。
たとえば、Call フィールドに DJ1? と入力して **Enter** を押して送信すると、DJ1 で始まる局のみが応答します。

局はコールサインの部分一致を既に認識していることにご注意ください。たとえば、1〜2 文字だけ誤入力しても、DX 局は正しいコールサインを再度送信しようとします。


## キーボード
Web Morse Runner の主な操作はキーボード経由です。
### ファンクションキー
ファンクションキー F1〜F8 はさまざまなメッセージの送信に使用されます。割り当てられたメッセージは固定されており、メイン画面に表示されます。
しかし、モダンな OS はファンクションキーを音量やディスプレイの輝度調節などさまざまな目的で使用しています。ファンクションキーとして使用するには、OS やキーボードの設定を行う必要があるかもしれません。


#### macOS Ventura 以降
1) Apple メニュー -> システム設定 を選択。
2) サイドバーでキーボードをクリック。
3) 右側のキーボードショートカットボタンをクリック。
4) サイドバーでファンクションキーをクリック。
5) "F1、F2 などのキーを標準のファンクションキーとして使用" をオンにする。

#### Windows 11
Windows でファンクションキーを有効にする方法はさまざまです。著者のマシンでは Fn キーがサポートされており、Fn キーと "ロックシンボル" のキー (Esc キー上) を押すことで、ファンクションキーを永続的にロックできました。他にも BIOS や UEFI の設定で動作するという報告や、コントロールパネルのロックキーを使用する報告があります。


#### Chrome と F7
Google Chrome ブラウザでは、**F7** キーが "Caret Browsing" (キャレットブラウズ) の起動に使用されます。
Web Morse Runner の著者は Caret Browsing を使用していないため、単にオフにして「次回から確認しない」にチェックを入れています。そうすると **F7** キーは疑問符の送信に使用されます。


#### 数字キーの使用
モバイルなど、ファンクションキーの使用が難しいプラットフォームもあります。ファンクションキーの代わりに、CTRL-1 / Meta-1 / Alt-1 / Numpad 1 で F1、CTRL-2 / Meta-2 / Alt-2 / Numpad 2 で F2 などを押すことができます。

### サポートされるキーボードショートカット

#### ESM (Enter to Send Message)
Web Morse Runner の主な使用法は ESM 経由です。
これは、コンテスト開始後にカーソルが自動的に Call フィールドに置かれることを意味します。

* CQ を呼ぶには **Enter** キーを押すだけです。
* 局が応答したら、カーソルが既にある Call フィールドにコールサインを入力します。
* 再度 **Enter** を押します。Web Morse Runner はレポート 599 とランニングナンバーを送信します。カーソルは自動的に NR フィールドへ進みます。
* 相手のナンバーを聞いて、カーソルが既に進んでいる NR フィールドにそのナンバーを入力します。
* 再度 **Enter** を押すと、Web Morse Runner は **TU** を送信して QSO を完了します。
* QSO が完了すると、Call、RST、NR フィールドが自動的にクリアされ、カーソルは再び Call フィールドに置かれます。これで次の QSO に進むことができます。

#### その他のキーボード操作

* **TAB**: Call -> RST -> NR フィールド間でカーソルを移動します。NR フィールドにカーソルがあるときに再度 TAB を押すと Call に戻ります。
* **Space**: スペースを押すと、フォーカスされているフィールドが進みます (Call から NR、 RST から NR、NR から Call)。RST が空の場合、599 が自動的に入力されます。
* **Alt-Enter, Meta-Enter, Ctrl-Enter**: QSO を保存します。
* **Alt-W, Ctrl-W**: 入力フィールドをワイプ (クリア) します。
* **<;>, <Insert>**: F5 + F2 と同等です。
* **<+>, <.>, <,>, <[>**: F3 + Save と同等です。
* **<PageUp>, <PageDown>**: キーイング速度を 5 WPM 刻みで変更します。

* **ファンクションキー**
    * **F1**: CQ を呼ぶ
    * **F2**: RST とナンバーを送信: DX 局が NR? または AGN と聞いてきたときに使用します。
    * **F3**: TU を送信: DX 局に QSO の終了を伝えます。交信をすでにログに記録したものの、局が以前の TU を聞いていなかった可能性がある場合に必要になることがあります。
    * **F4**: 自分のコールを送信: 実際には Web Morse Runner ではめったに使用されません。
    * **F5**: DX のコールを送信: 局がコールを理解しなかった場合 (たとえば局と重なった場合) に非常に便利です。再びコールを送って、局に RST と NR を送信させることができます。
    * **F6**: B4 - 以前その局と交信済みです。Web Morse Runner ではめったに使用されません。
    * **F7**: 疑問符。コールを完全に取得できなかった場合、疑問符を送信すると相手局がコールを再度送ってくれます。
    * **F8**: NIL - 相手局に相手のコールを取得できなかったことを伝えます。間違ったコールサインをすでに送信してしまった場合に Web Morse Runner で非常に便利です。DX 局はあなたを他の誰かと交信したものと判断し、応答しなくなります。NIL を送信することで相手のコールを取れなかったことを示し、DX 局は自分のコールを再度送信しようとします。
* **Arrow-Up/Down**: RIT を制御します。局は常に同じ周波数で呼び出してはくれません。フィルタ帯域幅によっては、DX 局が聞こえなかったり、非常に弱い信号であったりします。矢印キーを使って RIT を上下に移動してください。
* **CTRL-Arrow-Up/Down, Alt-Arrow-Up/Down, Meta-Arrow-Up/Down**: 帯域幅。
* **Escape**: 送信を中断します。

## 統計情報
Web Morse Runner は (バージョン 0.11-Beta 以降) 統計情報を提供します。データはログされたデータに基づいており、確認済み・未確認の QSO を含みます。
QSO/h は最初のコンテストをログした後表示されます。
5 分のマーカーに達すると、過去 5 分間の QSO データが反映されます。
Web Morse Runner は 5 分間隔の QSO/h を棒グラフでも表示します。24 本のバー (各 5 分) で構成されています。これは長時間のコンテストセッションを行う場合に便利です。

## トランスクリプト
トランスクリプトにはコンテスト中に交換されたすべてのメッセージが時系列で表示されます。デフォルトでは非表示になっています。**📄 (ページアイコン)** をクリックして表示/非表示を切り替えます。新しいコンテストを開始するとトランスクリプトはクリアされます。

## エキスパート設定
**⚙ (歯車アイコン)** をクリックしてエキスパート設定パネルを開きます。これらの設定はコンテスト実行中も変更すると即座に反映されます。

* **Max Pile-up**: パイルアップ関連モードで同時に呼び出してくる DX 局の最大数を制限します。より管理しやすい呼び出し局数で練習したい場合に便利です。0 に設定すると無制限になります。
* **DX Stations WPM**: 呼び出し DX 局の速度を制御します:
    * *Standard* — 局はあなたの CW 速度設定の半分から全速の範囲で応答します。
    * *Individual* — 自分の速度とは別に、DX 局の WPM 範囲の最小値と最大値を独自に設定します。
* **Min WPM / Max WPM** (Individual モードで表示): DX 局の速度範囲の下限と上限。
* **Farnsworth Timing**: 有効にすると、文字は完全な文字速度で送信されますが、文字と単語の間に余分な間隔が加わり、実効 WPM レートが低くなります。これは CW を速度で学習するための古典的なファーンズワース方式です。実効 WPM は **Eff. WPM** フィールドで設定します — あなたの CW Speed より低くする必要があります。
* **Contest Start Offset**: すでに一定時間進行しているコンテストに参加することをシミュレートします。DX 局のシリアルナンバーはオフセットに対応する値から始まるため、常に 001 から始めるのではなく、より現実的な交換ナンバーの練習に役立ちます。

## WAV 録音
Web Morse Runner は生成された音声を WAV ファイルに録音し、後でレビューや分析に使用できます。

**⏺ Rec** ボタンはコンテストパネルで常に利用可能です。Run ボタンとは独立して動作します:

1. **録音の待機 (Arm)** — コンテストの開始前または最中に **⏺ Rec** を押します。ボタンが赤色になり、待機中で準備ができていることを示します。
2. **録音中** — コンテストが実行中でボタンが待機状態であれば、録音が自動的に開始されます。ボタンが点滅し、音声がキャプチャ中であることを示します。
3. **コンテスト間で一時停止** — 待機中にコンテストが停止すると、ボタンは赤色の点灯に戻ります。次のコンテストが始まると、録音は同じセッションに追加するかたちで自動的に再開されます。
4. **ダウンロード** — 再度 **⏺ Rec** を押して待機状態を解除します。録音が停止し、WAV ファイルが自動的にダウンロードされます。

出力は非圧縮 PCM WAV ファイル (11,025 Hz、16-bit モノラル) で、ファイル名は `morse_<CALL>_<DATE>_<TIME>Z.wav` です。このサンプルレートでは CW トーンに適しており、ファイルサイズは約 1.3 MB/分です。1 セッションあたりの録音は 90 分 (約 119 MB) で上限に達します。

  * 新しい **IARU VHF コンテスト** モード。交換情報は RST、シリアルナンバー、および自分自身の QTH ロケータ (LOC) です。プルリクエストを寄贈してくださった Kalin (LZ1MZK) 氏に感謝します。[Example_Calls](Example_Calls) フォルダに 2 つの新しいサンプルコールファイルが追加されました: `IARU-VHF-NOCW.txt` (混合モードの局) と `IARU-VHF-ONLYCW.txt` (CW 運用局)。どちらのファイルもコールサインを 6 文字の Maidenhead ロケータと関連付けています。
* **0.17.1-beta** (2026-06-10)
  * バグ修正: マイナーなレイアウトバグを修正。
* **0.17-beta** (2026-06-09)
  * WAV 録音: **⏺ Rec** ボタンは Run ボタンとは独立してレコーダーを待機状態にします。コンテストの実行中に音声をキャプチャし、ボタンを再度押すと WAV ファイルとしてダウンロードされます。録音の待機/解除は複数のコンテストセッションをまたいでもデータが失われません。
* **0.16-beta** (2026-06-06)
  + エキスパート設定の一部として Farnsworth タイミングをサポートする新しい設定。
  * より大きなコンテストナンバーをサポートする新しい設定。
* **0.15-beta** (2026-06-02)
   * コンテスト設定内の「歯車」アイコンを押して開く新しいエキスパート設定。
       * Max Pile-up: パイルアップ関連モードで応答する DX 局の数を制限し、あまりにも多くの局が呼んでくる状況を避けることができます。
       * DX-Stations: Morserunner の標準設定 (デフォルト) を使用するか、応答する DX 局の Max/Min WPM を個別に設定できます。標準設定では、DX 局の速度はあなたの速度の最小値の半分からあなたの速度の最大値までの範囲になります。
* **0.14-beta** (2026-06-01)
   * 機能要望に沿ってトランスクリプトを追加しました。このトランスクリプトはデフォルトでは非表示ですが、「ページ」アイコンをクリックすることで表示できます。
* **0.12-beta** (2026-05-24) -- **Pentecost 2026 Edition**
   * 新しい **CWOps CWT Contest** を追加。David - CT7AUP のプルリクエストに感謝します。このプルリクエストは現在のアーキテクチャにうまく適合するように修正されました。GitHub のバグレポートを歓迎します。
* **0.11.2-beta** (2026-05-03)
   * バグ修正: Apple キーボードでは option キーと数字の組み合わせで対応するファンクションキーを発行できますが、<Option>-5 で <His> ではなく <His><TU> が送信されていました。これはブラウザのイベント処理が key 属性に "[" を送信するためです。OE8ZZZ からのご報告、フィードバックに感謝します。
   * DARC-Ausbildungscontest および A1Club Contest で正しいログ記録を妨げる可能性のある小さなリグレッションを削除しました。
* **0.11.1-beta** (2026-01-01)
   * バグ修正: <Insert> または <;> で Call / Nr を送信する際は、送信済みの Call/Nr を記憶しておき、ESM を使用した際にその Call/Nr を使用するようにしました。
* **0.11-beta** (2025-12-27) -- **Zwischen den Jahren 2025 Edition**
   * 新機能: QSO per hour をテキストとグラフで表示。
   * 新機能: WPX コンテストモード。(バンドコンディションは強制されません)
   * qso/h チャートのスペースを確保するための UI 再設計。
   * バグ修正: QRN バンドコンディション使用時に正弦波トーンが発生する問題 (フィルタ内の振幅カーブが平坦になる)。
   * バグ修正: コンテスト実行中に QSB/Flutter バンドコンディションを有効にできない問題。
* **0.10-beta** (2025-12-20) -- **Christmas 2025 Edition**
   * バグ修正: F/DJ1TF のような CEPT プリフィックスでプリフィックス検出が機能するように修正。
   * より多くのキーボードショートカットをサポート (リクエストしてくれた DK5TX と DJ5CW に感謝)
      * Alt-W, Ctrl-W, F11 - 入力フィールドをワイプ。
      * Alt-Enter, Shift-Enter, Ctrl-Enter - QSO を保存。
      * <+>, <.>, <,>, <[> - F3 + Save と同等
      * <;>, <Insert> - F5 + F2 と同等
      * <PageUp>, <PageDown> - WPM を 5 ずつ変更
      * <CTRL>-Arrow-Up/Down - 帯域幅
   * バグ修正 (2025-12-20):
      * DK5TX のバグ報告に感謝: コールが変更され、<Insert> または <;> が押されると、コールが送信され QSO が DX 局に確認されます。
* **0.9-beta** (2025-12-06) - **Saint Nicholas Day Edition**
   * Escape キーを押すと送信が中断される
   * TX インジケーター
   * 送信開始後にコールサインを更新することが可能で、その更新は変更がまだ送信されていない場合にのみ成功します。
* **0.8.4-beta** (2025-12-03)
   * Safari と Chrome でオーディオプロセスが開始されない場合がある macOS のいくつかの状況を改善。
   * macOS 上の Firefox では 48,000 kHz を超える出力サンプルレートが問題を引き起こす可能性があります。サンプルレートを適宜下げるか、Chrome または Safari を使用することをお勧めします。
* **0.8.3-beta** (2025-12-02)
   * Lutz DM6EE から報告されたバグを修正。スペースキーが機能しませんでした。これは Android 対応のために導入されたリグレッションでした。バグレポートに感謝します。
   * 初期化中にまれに発生するタイミングバグを修正。
* **0.8.2-beta** (2025-05-28)
   * 部分一致のコールサインをマッチする際、オリジナルの Morse Runner ほど厳格ではありませんでしたが、Web Morse Runner も緩和されました。
* **0.8.1-beta** (2025-05-27)
   * 不完全なコールサインで呼び出した後、局が応答しない原因となる 2 つのバグを修正。
* **0.8-beta** (2025-05-21)
   * HST モードを追加、grazie Pietro IN3GYO
   * 小さなバグ修正
   * (2025-05-22) バグ修正:
     * ファンクションキー F5 が <his> を送信しない。
* **0.7-beta** (2025-04-08) - **Easter Edition**
   * 異なるコンテストを可能にするためのコードの内部再構築をさらに進めました。
   * バグ修正: 設定がまだ初期状態の場合にコンテスト定義が正しく読み込まれない場合がありました。Firefox ブラウザで発見されました。Chrome ベースのブラウザはモジュールをわずかに異なる方法でインスタンス化するようです。
   * AWT (A1Club Weekly Contest) および DARC CWA コンテストの試験的サポート
   * (2025-04-13) いくつかのバグを修正:
      * 元のリクエストが保留中の間に追加の DX 局がリクエストされる問題を回避。
      * 一部の Android デバイスは異なるキーボードイベントを使用します (報告してくれた TOM DG5CW に感謝)。
      * 新しいコンテスト定義の読み取りに関するバグにより、コンテスト開始時にコンテスト交換フィールドが非表示になる可能性がありました。
      * ファンクションキーの押下をシミュレートするための Alt 修飾子を追加。


* **0.6-beta** (2025-2-27) - **Carnival Edition**
   * 将来のバージョンで異なるコンテストを可能にするためのコードの内部再構築
   * Tom (DF7TV) が JJ1WTL のデータベースに基づいた日本のコールサインのリストを提供しました。(JA-All_JJ1QTL ファイル)
   * DL6ER のデータベースに基づくドイツのコールのリストが提供されています (DL-All_DOK.txt ファイル)
   * モバイル使用: CTRL または Meta キーを押し続けると、ファンクションキーの代わりに数字キーを使用できます。(一部のモバイルプラットフォームはファンクションキーをサポートしていません)。テンキーを備えたキーボードを使用している場合は、これらのキーも使用できます。
* **0.5-beta** (2024-12-27) - **お正月 / New Year 2024 Edition**
   * バンドコンディション QRN、QRM、QSB、Flutter、LID のサポートを追加。
   * 新しいキーボードショートカット: **shift** キーがサポートされました。
   * コールサインがブラウザのローカルストレージにキャッシュされ、再ロードされなくなりました。
   * 新しい試験的機能: 独自のコールファイルをアップロード。(K5GQ 寄贈のサンプルコールリスト)
   * いくつかの重要なバグ修正。
* **0.4-alpha** (2024-12-15) -- **Christmas 2024 Edition**
   * パイルアップモードを追加。
   * Tab キーのシーケンスを Call/RST/NR フィールドに限定。
   * Call/RST/NR フィールドのサイズを拡大
   * さまざまなバグを修正。
* **0.3-alpha** (2024-12-12) バグ修正: v0.2-alpha の修正を含むモールスタイミングを修正。
* **0.2b-alpha** (2024-12-11) 0.2 での変更は、2 つのメッセージが続けて送信される際に audible gap が発生するリグレッションを引き起こしました。これはオーバーリアロケーションが原因でした。audible gap を避けるために v0.2-alpha の変更をロールバックしました。
* **0.2-alpha** (2024-12-11) 改善/バグ修正: この JavaScript バージョンは動的に割り当てられた Array を使用していましたが、Pascal では静的にサイズ指定された Array が使用されていました。これを Float32Array に事前割り当てされた配列に変更し、オーディオバッファプロセスでの不要な割り当てを避けることでパフォーマンスも改善しました。状況によっては間隔が広がる可能性がありました。
